How to read SQL Server execution plan if I want to know CPU usage on each query execution
## Bing
Reading a SQL Server execution plan can provide valuable insights into the CPU usage of each query execution. Here are some steps to do so:
- Execution Plan: Generate the execution plan of your query. You can do this by running your query in SQL Server Management Studio and clicking on the "Include Actual Execution Plan" button.
- Plan Diagram: In the plan diagram, you can right-click and change to "Per Node Costs". Then right-click again, and choose "Costs By > CPU." This will show you the cost percentages for operators based on overall CPU usage.
- Operator Costs: Each operator in a SQL Server execution plan is associated with a cost. The operator cost is relative to other costs in the execution plan. Usually, you need to concentrate on the costly operator and tuning the query around it.
- Performance Dashboard: Once you connect to your SQL Server or Azure SQL instance, you can select Reports > Performance Dashboard and see the current and historical values of CPU usage.
Remember, interpreting query plans correctly is the first and major principle to troubleshoot query performance issues. By understanding how to read an execution plan, you can gain practical experience that helps solve query performance issues.

what does the cpu cost unit? is it percentage?
## Bing
The "Estimated CPU Cost" shown in the SQL Server execution plan does not represent the estimated CPU usage percentage. It is a unitless number generated by the cost model. When added to the IO Cost and other fixed per operator cost, this gets an overall plan cost that was originally somewhat correlated with execution time in seconds on a certain Microsoft employee's machine.
These days, it cannot be correlated with any specific estimate of CPU utilization as a percent or in elapsed time and is only intended to be used by the optimizer itself when costing plans and comparing plan costs. The CPU costs for the same operator will be largely the same across computers running the same version of SQL Server irrespective of the model of CPU that they contain.
So, it's important to note that this "Estimated CPU Cost" is not a direct measure of CPU usage in terms of percentage or time, but rather a relative measure used for query optimization and comparison.
