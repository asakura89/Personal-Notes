It seems like you are trying to use a keyless entity type in Entity Framework Core, which is an entity type that does not have a key property. A key property is a property that uniquely identifies each instance of an entity type. Keyless entity types are useful for mapping to database objects that do not have a primary key, such as views, tables without a primary key, or queries defined in the model.
To use a keyless entity type, you need to call the HasNoKey method in the OnModelCreating method of your DbContext. This method tells Entity Framework Core that the entity type does not have a key and should be treated as read-only. For example, if you have a keyless entity type called BlogPostsCount that maps to a view that counts the number of posts for each blog, you can configure it as follows:

protected override void OnModelCreating(ModelBuilder modelBuilder)
{
    modelBuilder.Entity<BlogPostsCount>()
        .HasNoKey()
        .ToView("View_BlogPostsCount");
}

The ToView method specifies the name of the database object that the entity type maps to. If you omit this method, Entity Framework Core will use the name of the entity type as the default name of the database object.

If you want to create an entity that uses a key, you need to configure a property that will serve as the primary key of the entity. A primary key is a property that uniquely identifies each instance of an entity type. You can use data annotations or fluent API to configure a primary key. For example, if you have an entity called Book and you want to use the property ISBN as the primary key, you can do it as follows:

// Using data annotations
public class Book
{
    [Key]
    public string ISBN { get; set; }
    public string Title { get; set; }
    public string Author { get; set; }
}

// Using fluent API
protected override void OnModelCreating(ModelBuilder modelBuilder)
{
    modelBuilder.Entity<Book>()
        .HasKey(b => b.ISBN);
}

You can also use a composite key, which is a combination of two or more properties that uniquely identify an entity instance. For example, if you have an entity called OrderDetail and you want to use the properties OrderId and ProductId as the composite key, you can do it as follows:

// Using data annotations
[PrimaryKey(nameof(OrderId), nameof(ProductId))]
public class OrderDetail
{
    public int OrderId { get; set; }
    public int ProductId { get; set; }
    public int Quantity { get; set; }
    public decimal UnitPrice { get; set; }
}

// Using fluent API
protected override void OnModelCreating(ModelBuilder modelBuilder)
{
    modelBuilder.Entity<OrderDetail>()
        .HasKey(od => new { od.OrderId, od.ProductId });
